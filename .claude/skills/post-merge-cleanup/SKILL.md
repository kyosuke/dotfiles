# マージ後の後処理

PRをマージしたあと、ローカルをデフォルトブランチ（main など）に戻して最新を取り込み、役目を終えたフィーチャーブランチを安全に片付ける。

**安全の要**: ブランチ削除は必ず `git branch -d`（小文字）を使う。`-d` は対象がデフォルトブランチに完全にマージされていなければ削除を拒否する。これが「ちゃんとマージされていなければ消さずに知らせる」ガードになる。`-D`（強制削除）は使わない。

## 対象ブランチの決定

引数でブランチ名が渡されていればそれを対象にする。なければ次で決める。

- 現在フィーチャーブランチ（デフォルトブランチ以外）にいる → そのブランチが対象
- すでにデフォルトブランチにいる → 後述の「まとめて片付け」に進む

デフォルトブランチは次で判定する（取れなければ `main`、なければ `master` を試す）:

```bash
git symbolic-ref --quiet refs/remotes/origin/HEAD | sed 's@^refs/remotes/origin/@@'
```

## 手順

1. 作業ツリーがクリーンか確認する。未コミットの変更があれば、ここで止めてユーザーに知らせる（pullやブランチ切り替えで失う恐れがあるため、勝手に stash しない）:

```bash
git status --short
```

2. デフォルトブランチに切り替える:

```bash
git checkout <default-branch>
```

3. fast-forwardでpullする（マージコミットを勝手に作らない）:

```bash
git pull --ff-only
```

`--ff-only` が失敗する場合はローカルとリモートが分岐している。ブランチは消さず、状況をユーザーに伝える。

4. 対象ブランチを安全削除する:

```bash
git branch -d <target-branch>
```

- 成功 → マージ済みと確認できた。手順5へ
- 失敗（`not fully merged`）→ デフォルトブランチにまだ取り込まれていない。**削除せず**、その旨をユーザーに伝える。squash/rebaseマージの場合もここに該当しうるので、必要なら `gh pr view <branch> --json state,mergedAt` でPRのマージ状態を確認し、本当にマージ済みなら `git branch -D` での削除可否をユーザーに確認する（自動では `-D` しない）

5. リモート追跡参照のうち、削除済みリモートブランチへの参照を掃除する:

```bash
git remote prune origin
```

6. 結果を報告する（戻ったブランチ・取り込んだ変更の有無・削除したブランチ／削除しなかった理由）。

## まとめて片付け（すでにデフォルトブランチにいる場合）

引数がなく、起動時点でデフォルトブランチにいるときは、マージ済みのローカルブランチを列挙して片付ける:

```bash
git checkout <default-branch>
git pull --ff-only
git branch --merged | grep -vE '^\*|^\s*(main|master)$'   # マージ済みブランチ一覧
```

列挙された各ブランチを `git branch -d` で削除し、`git remote prune origin` を実行する。1件も該当しなければ「片付け対象なし」と報告する。`git branch --merged` はデフォルトブランチに完全にマージされたものだけを挙げるので、未マージは自然に除外される。

## 注意

- `-D`（強制削除）は自動では使わない。`-d` が拒否したら、それは安全装置が働いた合図として扱う
- 未コミットの変更がある状態では勝手に進めない
- デフォルトブランチ自身は削除候補から必ず除外する
