# マージ後の後処理

PRをマージしたあと、ローカルをデフォルトブランチ（main など）に戻して最新を取り込み、役目を終えたフィーチャーブランチを安全に片付ける。

## 実行方法（Sonnetサブエージェントへ委譲）

この後処理はgitの機械的操作だけで、会話の文脈を必要としない。呼び出し元セッションでインライン実行すると、積み上がった会話履歴を約5回のツール往復すべてで再送してしまう（とくにこのスキルは作業末尾・`/clear`直前に走るため会話が一番重いタイミングになる）。そこで手順の実行はサブエージェントに丸投げし、呼び出し元は結果を取り次ぐだけにする。

- サブエージェント起動ツール（Agent / Task）で下記の手順を委譲する。`model: sonnet`、`subagent_type: general-purpose` を指定する。判断を要さない定型作業なのでSonnetで十分で、単価と分離の両方でトークンを節約できる
- 引数でブランチ名が渡されていれば、その値をサブエージェントのプロンプトに含めて渡す
- サブエージェントはユーザーに直接質問できない。だから停止条件（作業ツリーが汚れている・`--ff-only`が失敗・デフォルトブランチへのマージを確認できない）に当たったら、**勝手に判断せず状況を報告して止まる**よう指示する。`-D`での強制削除も`stash`もサブエージェントには絶対にさせない
- サブエージェントの報告をそのままユーザーに取り次ぐ。停止条件で止まっていたら、呼び出し元がユーザーに状況を伝えて指示を仰ぎ、必要なら判断を反映して再度委譲する

呼び出し元は、この段落より下の内容（次の「安全の要」からファイル末尾まで）を、そのままサブエージェントへの指示（プロンプト）として渡す。

**安全の要**: ローカルブランチを削除する前に、対象がデフォルトブランチにマージ済みかを自分で明示的に確認する。判定は `git merge-base --is-ancestor <対象> <デフォルト>` で行い、終了コードが0（対象がデフォルトブランチの先祖＝マージ済み）のときだけ削除する。削除コマンドは `git branch -d`（小文字）を使い、`-D`（強制削除）は使わない。

`git branch -d` 自体の「未マージなら拒否」ガードには頼らない。`-d` はupstream追跡ブランチが設定されていると、デフォルトブランチではなくupstream（`origin/<同名>`）との比較でマージ判定する。`push -u`済みのブランチはデフォルトブランチに未マージでも `-d` が通ってしまうため、`-d`単独では安全装置にならない。マージ確認は上記の `merge-base --is-ancestor` で行う。

## 対象ブランチの決定

引数でブランチ名が渡されていればそれを対象にする。なければ次で決める。

- 現在フィーチャーブランチ（デフォルトブランチ以外）にいる → そのブランチが対象
- すでにデフォルトブランチにいる → 後述の「まとめて片付け」に進む

デフォルトブランチは次で判定する（取れなければ `main`、なければ `master` を試す）:

```bash
git symbolic-ref --quiet refs/remotes/origin/HEAD | sed 's@^refs/remotes/origin/@@'
```

## 手順

1. 作業ツリーがクリーンか確認する。未コミットの変更があれば、ここで止めて状況を報告する（pullやブランチ切り替えで失う恐れがあるため、勝手に stash しない）:

```bash
git status --short
```

2. デフォルトブランチに切り替える:

```bash
git checkout <default-branch>
```

3. fast-forwardでpullする（マージコミットを勝手に作らない）:

```bash
git pull --ff-only
```

`--ff-only` が失敗する場合はローカルとリモートが分岐している。ブランチは消さず、状況を報告して止まる。

4. 対象ブランチがデフォルトブランチにマージ済みかを確認してから削除する。まず先祖判定を行う:

```bash
git merge-base --is-ancestor <target-branch> <default-branch>; echo $?
```

- `0`（デフォルトブランチの先祖＝マージ済み）→ `git branch -d <target-branch>` で削除し、手順5へ。マージは確認済みなので通常は `-d` も成功する。例外はローカルがupstreamより先行しているとき（デフォルトブランチには入っているが、ローカルの一部コミットが `origin/<同名>` に未push）で、`-d` はupstream基準で判定するため拒否することがある。この拒否はマージ未確認とは別物なので、`-D` は使わず、upstream先行で `-d` が拒否された旨を報告して呼び出し元に制御を戻し、止まる
- `1`（先祖でない）→ 通常マージではまだ取り込まれていない。ただしsquash/rebaseマージではマージ済みでも先祖にならないので、`gh pr view <branch> --json state,mergedAt` でPRのマージ状態を確認する:
  - PRがマージ済み（`state`が`MERGED` / `mergedAt`が非null）→ squash/rebaseで取り込まれている。この場合 `git branch -d` は先祖でないため拒否し、削除には `-D` が要る。サブエージェントは `-D` を実行せず、PRマージ済みである旨と `-D` の可否をユーザーに確認する報告をして、呼び出し元に制御を戻して終了する
  - PRが未マージ（`state`が`OPEN`など）→ 本当に未マージ。**削除せず**、その旨を報告して止まる

5. リモート追跡参照のうち、削除済みリモートブランチへの参照を掃除する:

```bash
git remote prune origin
```

6. 結果を報告する（戻ったブランチ・取り込んだ変更の有無・削除したブランチ／削除しなかった理由）。

## まとめて片付け（すでにデフォルトブランチにいる場合）

引数がなく、起動時点でデフォルトブランチにいるときは、マージ済みのローカルブランチを列挙して片付ける:

```bash
git checkout <default-branch>
git pull --ff-only
git branch --merged <default-branch> | grep -vE '^\*|^\s*(main|master)$'   # マージ済みブランチ一覧
```

列挙された各ブランチを `git branch -d` で削除し、`git remote prune origin` を実行する。1件も該当しなければ「片付け対象なし」と報告する。`git branch --merged <default-branch>` は引数のコミット（デフォルトブランチ）から到達できるブランチだけを挙げる。upstreamではなくデフォルトブランチとの比較なので、`git branch -d` 単独の判定と違い未マージを取り違えない。squash/rebaseマージのブランチは先祖にならず列挙されないため、この一括経路では消えずに残る（個別の手順4で扱う）。

## 注意

- `-D`（強制削除）は自動では使わない。`merge-base --is-ancestor` でマージを確認できないときは削除しない
- マージ判定は `git branch -d` の内蔵ガードに頼らない。upstream追跡があると判定基準がデフォルトブランチではなくupstreamになり、`push -u`済みの未マージブランチでも `-d` が通ってしまう
- 未コミットの変更がある状態では勝手に進めない
- デフォルトブランチ自身は削除候補から必ず除外する
- サブエージェントは停止条件でユーザーに尋ねず、状況を報告して止まる。ユーザーとのやり取りは呼び出し元セッションが担う
