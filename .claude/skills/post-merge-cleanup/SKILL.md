# マージ後の後処理

PRをマージしたあと、ローカルをデフォルトブランチ（main など）に戻して最新を取り込み、役目を終えたフィーチャーブランチを安全に片付ける。

## Sonnetサブエージェントへ委譲する

gitの機械的操作だけで会話の文脈を要さない後処理を、呼び出し元セッションでインライン実行すると、積み上がった会話履歴を毎ツール往復で再送してしまう（このスキルは作業末尾・`/clear`直前に走り、会話が最も重いタイミングになる）。手順の実行はサブエージェントに丸投げし、呼び出し元は結果を取り次ぐだけにする。

- Agent / Task で `model: sonnet`、`subagent_type: general-purpose` を指定して委譲する。定型作業なのでSonnetで十分で、単価と分離の両方でトークンを節約できる
- 引数でブランチ名が渡されていれば、サブエージェントのプロンプトに含める
- サブエージェントはユーザーに直接質問できない。停止条件（作業ツリーが汚れている・`--ff-only`失敗・デフォルトブランチへのマージを確認できない）に当たったら、勝手に判断せず状況を報告して止まるよう指示する。`-D`での強制削除も`stash`も絶対にさせない
- サブエージェントの報告をそのままユーザーに取り次ぐ。停止していたら呼び出し元がユーザーに指示を仰ぎ、必要なら判断を反映して再委譲する

呼び出し元は、次の「安全の要」からファイル末尾までを、そのままサブエージェントへの指示として渡す。

## 安全の要

ローカルブランチを削除する前に、対象がデフォルトブランチにマージ済みかを `git merge-base --is-ancestor <対象> <デフォルト>` で確認し、終了コードが0（マージ済み）のときだけ削除する。削除は `git branch -d`（小文字）を使い、`-D`（強制削除）は使わない。

マージ判定を `git branch -d` の内蔵ガードに頼らないこと。`-d` はupstream追跡ブランチがあるとデフォルトブランチではなくupstream（`origin/<同名>`）と比較して判定するため、`push -u` 済みのブランチはデフォルトブランチに未マージでも `-d` が通る。マージ確認は必ず `merge-base --is-ancestor` で行う。

## 対象ブランチの決定

引数があればそれを対象にする。なければ次で決める。

- フィーチャーブランチ（デフォルトブランチ以外）にいる → そのブランチが対象
- すでにデフォルトブランチにいる → 「まとめて片付け」に進む

デフォルトブランチは次で判定する（取れなければ `main`、なければ `master`）:

```bash
git symbolic-ref --quiet refs/remotes/origin/HEAD | sed 's@^refs/remotes/origin/@@'
```

## 手順

1. 作業ツリーがクリーンか確認する。未コミットの変更があれば止めて報告する（pullや切り替えで失う恐れがあるため勝手にstashしない）:

```bash
git status --short
```

2. デフォルトブランチに切り替え、fast-forwardでpullする:

```bash
git checkout <default-branch>
git pull --ff-only
```

`--ff-only` が失敗するのはローカルとリモートが分岐しているとき。ブランチは消さず、報告して止まる。

3. 対象ブランチがマージ済みかを確認して削除する:

```bash
git merge-base --is-ancestor <target-branch> <default-branch> && echo 0 || echo $?
```

- `0`（マージ済み）→ `git branch -d <target-branch>` で削除し、手順4へ。例外として、デフォルトブランチには入っているがローカルの一部コミットが `origin/<同名>` に未pushだと、`-d` はupstream基準で拒否することがある。これはマージ未確認とは別物なので `-D` は使わず、upstream先行で拒否された旨を報告して止まる
- `1`（先祖でない）→ 通常マージでは未取り込み。ただしsquash/rebaseマージはマージ済みでも先祖にならないため、`gh pr view <branch> --json state,mergedAt` で確認する:
  - マージ済み（`state` が `MERGED` / `mergedAt` が非null）→ squash/rebaseで取り込み済み。`-d` は先祖でないため拒否するので、`-D` の可否をユーザーに確認する報告をして止まる（`-D` は実行しない）
  - 未マージ（`state` が `OPEN` など）→ 削除せず報告して止まる

4. 削除済みリモートブランチへの追跡参照を掃除する:

```bash
git remote prune origin
```

5. 結果を報告する（戻ったブランチ・取り込んだ変更の有無・削除したブランチ／しなかった理由）。

## まとめて片付け（すでにデフォルトブランチにいる場合）

引数がなく、起動時点でデフォルトブランチにいるときは、マージ済みのローカルブランチをまとめて片付ける:

```bash
git checkout <default-branch>
git pull --ff-only
git branch --merged <default-branch> | grep -vE '^\*|^\s*(<default-branch>|main|master)$'
```

列挙された各ブランチを `git branch -d` で削除し、`git remote prune origin` を実行する。該当なしなら「片付け対象なし」と報告する。`git branch --merged <default-branch>` はデフォルトブランチから到達できるブランチだけを挙げる（upstreamではなくデフォルトブランチとの比較なので未マージを取り違えない）。squash/rebaseマージのブランチは先祖にならず列挙されないため、この経路では残る（個別の手順3で扱う）。

## 注意

- `-D`（強制削除）は自動で使わない。`merge-base --is-ancestor` でマージを確認できないときは削除しない
- 未コミットの変更がある状態では勝手に進めない
- デフォルトブランチ自身は削除候補から必ず除外する
- サブエージェントは停止条件でユーザーに尋ねず、報告して止まる。ユーザーとのやり取りは呼び出し元が担う
