# Dual Review — Codex + Claude Code 並列コードレビュー

Codex CLI と Claude Code サブエージェントの2系統で同時にコードレビューを行い、結果を統合してユーザーに提示する。

## 手順

### 1. レビュー対象の差分を確認

```bash
git diff --stat main...HEAD
```

`$ARGUMENTS` にベースブランチの指定がある場合（例: `develop`）はそれを使う。指定がなければ `main` をベースとする。

**差分がない場合の fallback**: `main...HEAD` に差分がなく、未コミットの変更（`git diff` や untracked files）がある場合は、未コミットの変更をレビュー対象にする。未コミットの変更もない場合のみ、ユーザーに確認する。

### 2. 並列レビューを実行

以下の2つを **同時に** 実行する（Agent ツールで並列起動）:

**Codex レビュー:**
```bash
# ブランチ差分がある場合
codex review --base <ベースブランチ>

# 未コミットの変更をレビューする場合（fallback 時）
codex review --uncommitted
```
- Bash の timeout を 300000 に設定すること
- 未コミットの変更をレビューする場合は `--base` ではなく `--uncommitted` フラグを使うこと
- Codex が停滞・タイムアウトした場合は結果なしとして扱い、Claude 側の結果のみで進める

**Claude Code サブエージェント:**
Agent ツールで subagent_type を指定せず（general-purpose）起動し、以下のプロンプトを渡す:

> このブランチの main からの差分をレビューしてください。
> `git diff main...HEAD` で差分を取得し、以下の観点で分析:
> - バグ・ロジックエラー
> - セキュリティ上の問題
> - パフォーマンス懸念
> - 型安全性の問題
> - テストカバレッジの欠落
>
> 各指摘を以下の形式で報告:
> - **severity**: critical / warning / note
> - **file**: ファイルパスと行番号
> - **issue**: 問題の説明
> - **suggestion**: 改善案（あれば）
>
> コードの変更は一切しないこと。レポートのみ。

### 3. 結果を統合してユーザーに提示

両方の結果を以下のルールで統合する:

- **重複排除**: 同じファイル・同じ問題を指している指摘はまとめる
- **出典明記**: 各指摘に `[Codex]` `[Claude]` `[両方]` のラベルをつける
- **severity 順**: critical → warning → note の順で提示
- **番号付きリスト**: ユーザーが「3番と5番を修正して」のように指定できるようにする

### 4. 修正の実行

統合結果をもとに、以下のように対応を分ける:

**自動修正する（ユーザー確認不要）:**
- 明らかなバグ・ロジックエラー
- セキュリティ上の問題
- 型エラー
- その他、直すべきことが明確な指摘

**ユーザーに判断を仰ぐ:**
- トレードオフがある変更（パフォーマンス vs 可読性など）
- 設計方針に関わる指摘
- レビュー指摘自体が妥当か判断が分かれるもの

修正時は:
- 指摘された箇所のみ変更する（無関係な変更をしない）
- レビュー結果を鵜呑みにせず、指摘が妥当か自分でも検証してから修正する
- 自動修正した内容と、判断を仰ぐ内容を分けて報告する
